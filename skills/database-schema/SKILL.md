## Conventions

| Convention   | Rule                                                                 |
|--------------|----------------------------------------------------------------------|
| Primary keys | UUID (`defaultRandom()`)                                             |
| Timestamps   | `created_at` with `defaultNow()`, `deleted_at` nullable             |
| Soft delete  | `isNull(table.deleted_at)` on reads, `set({ deleted_at: new Date() })` on deletes |
| Naming       | `snake_case` for all tables and columns                              |
| Foreign keys | `ON DELETE CASCADE` for neighborhood-scoped data                     |

### `neighborhood_members`

```ts
export const neighborhoodMembers = pgTable('neighborhood_members', {
  id: uuid('id').defaultRandom().primaryKey(),
  user_id: uuid('user_id').notNull().unique()
    .references(() => users.id, { onDelete: 'cascade' }),
  neighborhood_id: uuid('neighborhood_id').notNull()
    .references(() => neighborhoods.id, { onDelete: 'cascade' }),
  role: varchar('role', { length: 20 }).notNull(), // membro | admin | superadmin
  joined_at: timestamp('joined_at').defaultNow().notNull(),
  deleted_at: timestamp('deleted_at'),
});
```

**Rules:**
- `user_id` is UNIQUE — one neighborhood per user
- Exactly one `superadmin` per `neighborhood_id` (enforced at application level)

### `posts`

```ts
export const posts = pgTable('posts', {
  id: uuid('id').defaultRandom().primaryKey(),
  type: varchar('type', { length: 20 }).notNull(),        // aviso | negocio
  title: varchar('title', { length: 255 }).notNull(),
  description: text('description').notNull(),
  category: varchar('category', { length: 30 }),          // seguranca | utilidade_publica | eventos
  business_type: varchar('business_type', { length: 20 }), // produto | servico
  contact: varchar('contact', { length: 30 }),
  image_url: varchar('image_url', { length: 500 }),
  author_id: uuid('author_id').notNull()
    .references(() => users.id, { onDelete: 'cascade' }),
  neighborhood_id: uuid('neighborhood_id').notNull()
    .references(() => neighborhoods.id, { onDelete: 'cascade' }),
  active: boolean('active').notNull().default(true),
  created_at: timestamp('created_at').defaultNow().notNull(),
  expires_at: timestamp('expires_at').notNull(),
  deleted_at: timestamp('deleted_at'),
}, (table) => ({
  neighborhoodTypeIdx: index('posts_neighborhood_type_idx')
    .on(table.neighborhood_id, table.type, table.active),
  expiresIdx: index('posts_expires_idx').on(table.expires_at),
}));
```

**Rules:**
- `type = aviso` → `category` required
- `type = negocio` → `business_type` and `contact` required

### `statuses`

```ts
export const statuses = pgTable('statuses', {
  id: uuid('id').defaultRandom().primaryKey(),
  type: varchar('type', { length: 30 }).notNull(),
  // falta_agua | falta_energia | barulho_excessivo | coleta_lixo
  neighborhood_id: uuid('neighborhood_id').notNull()
    .references(() => neighborhoods.id, { onDelete: 'cascade' }),
  total_confirmations: integer('total_confirmations').notNull().default(0),
  active: boolean('active').notNull().default(true),
  started_at: timestamp('started_at').defaultNow().notNull(),
  expires_at: timestamp('expires_at').notNull(),
  deleted_at: timestamp('deleted_at'),
}, (table) => ({
  activeUniqueIdx: uniqueIndex('statuses_active_unique_idx')
    .on(table.neighborhood_id, table.type)
    .where(sql`active = true AND deleted_at IS NULL`),
}));
```

**Rule:** Only one active status per type per neighborhood (partial unique index).

```ts
export const userDevices = pgTable('user_devices', {
  id: uuid('id').defaultRandom().primaryKey(),
  user_id: uuid('user_id').notNull()
    .references(() => users.id, { onDelete: 'cascade' }),
  expo_push_token: varchar('expo_push_token', { length: 200 }).notNull().unique(),
  platform: varchar('platform', { length: 10 }).notNull(), // ios | android

```

**Rules:**
- `expo_push_token` is UNIQUE globally
- Hard delete on logout or `DeviceNotRegistered` error from Expo
- `last_seen_at` updated on every token upsert

## Common Query Patterns

### Soft delete read

```ts
.where(and(eq(table.neighborhood_id, id), isNull(table.deleted_at)))
```

### Soft delete write

```ts
await db.update(table).set({ deleted_at: new Date() }).where(eq(table.id, id));
```

### Upsert (devices)

```ts
await db.insert(userDevices)
  .values({ user_id, expo_push_token, platform, last_seen_at: new Date() })
  .onConflictDoUpdate({
    target: userDevices.expo_push_token,
    set: { user_id, platform, last_seen_at: new Date() },
  });
```

### Atomic counter increment

```ts
await db.update(statuses)
  .set({ total_confirmations: sql`${statuses.total_confirmations} + 1` })
  .where(eq(statuses.id, statusId));
```

### Safe counter decrement (floor at 0)

```ts
await db.update(statuses)
  .set({ total_confirmations: sql`GREATEST(${statuses.total_confirmations} - 1, 0)` })
  .where(eq(statuses.id, statusId));
```
